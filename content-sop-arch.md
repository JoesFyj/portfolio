# 内容创作 SOP 架构文档
> 版本：v1.0 | 日期：2026-05-05

---

## 一、SOP 流程总览

```
热点监控 → 选题筛选 → 内容生成 → AI评审 → 格式适配 → 发布追踪
              ↑                               │
              └────── 评审<8分打回（最多3轮）─────┘
```

**飞书三表协同**：
- 选题库：待审核选题池
- 文案库：已通过终审的成品
- 数据复盘：发布后各平台数据追踪

---

## 二、各环节详解

---

### 2.1 热点监控

**数据来源**：
| 平台 | 监控指标 |
|------|----------|
| 知乎 | 热榜 Top50、行业话题新增 |
| 微信公众号 | 爆款文章标题抓取 |
| 微博 | 热搜榜（每30分钟） |
| 抖音/小红书 | 关键词热度（AI/效率工具） |
| 即刻/知识星球 | 高赞讨论 |

**监控方式**：
- 半自动：RSS + 关键词告警（已有现成工具：Feeder、Notion Bot）
- AI 辅助：每日8点定时推送"今日热点摘要"到飞书群

**输出**：进入选题库待筛选

---

### 2.2 选题 Agent

**触发时机**：每日 8:00 AM 自动运行（或手动触发）

**输入**：
- 热点监控数据
- 个人定位：AI应用 · 全栈开发 · 效率工具
- 内容风格：实战干货 · 避免纯理论

**输出格式**：
```
{
  "topic": "标题候选",
  "angle": "切入角度（3个备选）",
  "target_platform": "首选平台",
  "hot_score": 1-10,
  "difficulty": "easy/medium/hard",
  "reasoning": "为什么选这个"
}
```

**Prompt 示例**：
> 你是一个资深自媒体内容策划，擅长在热点话题中找到适合AI开发者的切入角度。
>
> 背景：[今日热点数据]
> 定位：AI应用、全栈开发、效率工具
>
> 请输出3个选题，要求：
> 1. 有信息差，不是简单的热点复述
> 2. 普通人能做，有实操性
> 3. 角度新颖，避免同质化

---

### 2.3 内容 Agent

**核心流程**：
1. 根据选题生成大纲
2. 大纲确认后展开正文
3. 配图建议（Canva 模板）

**输出格式**：
```
{
  "title": "最终标题",
  "outline": ["H2标题列表"],
  "content": "完整正文（含小标题）",
  "hashtags": ["#标签1", "#标签2"],
  "cover_suggestion": "封面文案建议",
  "word_count": 1200,
  "reading_time": "5分钟"
}
```

**Prompt 模板**：
```
你是一个专注AI工具与效率的全栈开发者，写作风格：
- 语言简洁有力，避免废话
- 有代码示例时务必可用
- 结尾有行动号召

选题：[topic]
角度：[angle]
目标平台：[platform]

请生成一篇适合[platform]平台的完整文章，格式见输出格式。
```

---

### 2.4 评审 Agent

**评分维度**（每项1-10分）：
| 维度 | 权重 | 评估要点 |
|------|------|----------|
| 吸引力 | 25% | 标题是否吸引人、前3秒是否抓人 |
| 信息价值 | 30% | 是否有独特信息、干货密度 |
| 可读性 | 20% | 结构清晰、语言流畅 |
| 可执行性 | 15% | 读者能否真的跟着做 |
| 平台适配 | 10% | 长度/格式是否适合目标平台 |

**总分**：加权平均，保留1位小数

**决策规则**：
- ≥8.0 分：**通过** → 进入格式适配
- 6.0-7.9 分：**打回修改** → 给出具体修改意见，循环最多3次
- <6.0 分：**放弃** → 选题作废，更换选题

**评审 Prompt 示例**：
```
你是一个资深编辑，正在评审一篇自媒体文章。

请从以下5个维度打分（1-10分），最后给出加权总分和建议：

维度说明：
1. 吸引力（25%）：标题是否吸引人
2. 信息价值（30%）：是否有独特干货
3. 可读性（20%）：结构是否清晰
4. 可执行性（15%）：读者能否实操
5. 平台适配（10%）：长度格式是否合适

文章：[content]
目标平台：[platform]

输出格式：
- 各维度得分及理由
- 总分（1位小数）
- 通过/打回
- 如打回，给出具体修改意见
```

---

### 2.5 格式适配

不同平台的内容调整：

| 平台 | 格式调整 |
|------|----------|
| 微信公众号 | 配图、分割线、重点加粗、排版优化 |
| 知乎 | 问题导向开头、代码块、延伸阅读 |
| 小红书 | 封面文案、口语化、emoji、标签密集 |
| B站专栏 | 长文结构、参考资料、互动问题 |
| 即刻 | 短平快、碎片化、话题标签 |

**输出**：各平台适配版文案，存入文案库

---

### 2.6 发布追踪

**发布后记录**：
```json
{
  "work_id": "uuid",
  "platforms": ["微信公众号", "知乎"],
  "publish_time": "2026-05-05 10:00",
  "initial_likes": 0,
  "final_likes": 0,
  "comments": 0,
  "notes": "首次发布备注"
}
```

**数据复盘**：发布后7天汇总各平台数据，更新记录

## 六、关键原则

1. **选题大于写作**：好选题是爆款的前提，宁可多花时间选
2. **评审严格不将就**：<8分的宁可重来，质量比数量重要
3. **平台适配优先**：一篇通发效果差，针对性改编收益大
4. **数据驱动迭代**：每周复盘，数据差的分析原因，改进选题方向
5. **自动化减轻负担**：AI 能做的交给 AI，人工专注决策和审核
